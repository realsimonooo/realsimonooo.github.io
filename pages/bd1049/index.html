06.文件系统

# 1️⃣ 文件系统的概念

文件： 具有文件名的一组相关信息的集合；

文件系统： 与管理文件有关的软件和数据。由下三部分组成：

  • 管理文件所需要的数据结构；
  • 相应的管理软件；
  • 被管理的文件；

文件系统作用：对文件的存储空间进行组织、分配，负责文件的存储，并对文件进行保护、检索的系统。

# 文件的分类

按照用途分类：

  • 系统文件：操作系统核心和各种系统应用程序和数据；
  • 库文件：各种标准子程序库（C语言子程序库等）；
  • 用户文件：源程序、目标程序、用户数据库；

按照文件保护级别分类：

  • 只读文件：只允许查看的文件；
  • 读写文件：可查看和修改，但不能运行；
  • 执行文件：可以运行，完成特定的功能；
  • 不保护文件：不设防的文件，对使用、查看、修改不限制。

按照信息流向分类：

  • 输出文件；
  • 输入文件；
  • 输出/输入文件；

按照数据形式分类：

  • 源文件；
  • 目标文件；
  • 可执行文件；

# 3️⃣ 结构与存取方法

从用户的观点：文件的组织形式称为逻辑结构（用户使用的角度）；

从实现的观点：文件在外村上的存放格式称为文件的物理结构（系统存储角度）；

# 文件的逻辑结构

  • 字符流式的无结构文件：

    • 特点：查找基本信息单位比较困难；
    • 源程序、目标程序；

  • 记录式的有结构文件；

    • 连续结构：把记录按先生成的先后顺序连续排列的逻辑机构；

      特点：1）适用型强，可以用于所有文件；

      2）记录的排列顺序与记录的内容无关；

      3） 搜索能力差；

    • 多重结构： 把记录按照键和记录名排列成行列式结构；如：

      • N个记录n个键=维行列式，浪费存储空间；

      • 包含记录队列元素构成记录队列，m个队列构成了多重结构。

    • 转置结构：把含有相同键的记录指针全部指向该键；

      • 最适合根据给定键查找记录；

    • 顺序结构：按照给定的顺序排列规定形成顺序结构；

      • 有利于按照某种顺序来搜索、追加、删除记录；
        • 如按照时间作为关键字；

# 选取文件逻辑结构原则

  • 减少改动：修改时，尽量减少对已经存储好的文件信息进行改动；
  • 减少查找时间：在查找文件时尽量减少查找时间；
  • 占用存储空间最小：使文件信息占据最小的存储空间；
  • 便于用户操作；

# 文件的存取方法

  • 顺序存取法：按照文件的逻辑地址顺序进行存取；

  • 随机存取法：允许用户根据记录的编号来存取文件的任意一个记录；

  • 按键存取法：主要用于数据库管理系统中的存取方法；按照给定的键或者记录名进行；

    • 首先搜索到进行存取的记录的逻辑位置，再将其转换到相应的物理地址后进行存取；
    • 特点：
      • 最简单、直观；
      • 搜索时间与所搜索的表格大小的成正比；
      • 搜索效率低；
    • 搜索方法：线性搜索法、散列法、二分搜索法；

# 文件的物理结构

是文件在存储设备上的存放方法。决定了文件信息在存储设备上的存储位置，文件的物理地址决定了文件信息的逻辑地址到物理地址的转换；

常见的物理结构：

  • 连续文件：逻辑上连续的文件存放到物理块中。

    • 优点：逻辑块号到物理块号变换简单；

    • 缺点：要求连续的存储空间；不利于文件的动态扩充。

  • 串联文件：使用非连续的物理块存放文件信息，采用链表的方式将存放同一个文件的物理块链连接成串联队列；

    • 优点：只需知道第一块物理内存块的地址；适合修改的文件保存；

    • 缺点：搜索效率低；不适合随机存取；

  • 索引文件：要求系统为每个文件建立一个索引表，表中的每一栏目指出文件信息所在的逻辑块号和与之对应的物理块号。

    • 既可以满足文件的动态增长，又可以方便进行随机存取；

# 3️⃣ 文件的存储设备

顺序存储设备：只有在前面的物理块被存取访问过后，才能存取后续的物理块内容；

  • 磁带：最典型的顺序存取设备；
    • 如果带速高，信息密度大，所需块间隙小的情况下，磁带的存取速度和数据传输率高；

直接存取设备：允许直接存取磁盘上的任意物理；

  • 磁盘：最典型的直接存取设备；
存储设备 物理结构 存取方法 文件长度
磁盘 顺序文件 直接或顺序 固定
串联文件 顺序 可变、固定
索引文件 直接或顺序 可变、固定
磁带 顺序文件 顺序 固定

先来先服务（FCFS）：按照进程请求访问磁盘先后顺序进行调度；

最短寻道时间优先(SSTF)：按当前磁头所在磁道距离最近的请求作为下一次服务的对象；

扫描(SCAN)：先满足一个方向的所有请求，再满足所有反方向的请求，扫描到头，这样循环反复。（电梯算法）

循环扫描(CSCAN):规定磁头单向移动。如果自里向外移动时,当磁头移到最外磁道时立即又返回到最里面磁道。

# 5️⃣ 文件的目录管理

文件说明：把文件名和对该文件进行控制管理的管理信息，按照一定的逻辑结构存放到物理存储块的一个表目中；

文件目录：一个文件的文件说明信息；

管理文件目录：就是管理文件信息；

文件的组成：

  • 包括文件说明和文件体；

  • 文件说明也可以称为文件控制块 / FCB；

文件的目录结构分类：

  • 一级目录：最简单的目录结构；
  • 二级目录：分为主文件目录和用户文件目录；
  • 多级目录：树形目录结构；

一级目录读写处理：

​ 特点：

  • 各文件说明项处于平等地位，按连续结构和顺序结构存放；
  • 文件名不能重名，会产生命名冲突；
  • 搜索效率低下；

二级目录读写处理：

​ 特点：

  • 可以解决重命名问题和文件共享问题；
  • 搜索效率提高；

多级文件系统：

  • 层次清楚，便于管理和保护；
  • 解决文件重命名；
  • 查找搜索速度快；

# 6️⃣ 文件的共享

使用三种方法实现：

  • 绕道法

  • 链接法

  • 基本文件目录表BFD

绕道法：需要用户指定所要共享的文件的逻辑位置或者到达被共享文件的路径。搜索效率不高；

链接法：将一个目录中的链指针直接指向被共享文件所在的目录；仍然需要用户指定被共享文件和被链接目录。

文件目录分为两部分：

  • BFD / 基本文件目录表： 文件的结构信息、物理块号、存取控制和管理信息，由系统赋予唯一标识符；
    • 存放文件说明信息和文件内部标识符；
  • SFD / 符号文件目录表：用户给予的标识符和文件说明信息的内部标识符；
    • 存放文件名和文件内部标识符；

# 7️⃣ 文件的存取控制和保护

文件共享：不同用户共同使用一个文件；

文件保护：文件本身需要防止文件的用户对文件内容的破坏；

文件保密：文件未经拥有者许可，其他用户不可访问；

文件系统的存取控制部分应做到：

  • 对于拥有读、写、执行权限的用户，应允许对其操作；
  • 对于没有拥有读、写、执行权限的用户，应禁止对其操作；
  • 应防止一个用户冒充其他用户对文件进行存取；
  • 应防止用户误用文件；

      • 访问文件口令
    • 优点：占用内存少，执行时间短；
    • 缺点：保密性差，共享性差。

# 8️⃣ 文件的使用

文件系统以系统调用的方式为用户提供的服务：

  • 设置和修改用户对文件的存取权限
  • 建立、改变和删除目录的服务
  • 文件共享、设置访问路径的服务
  • 创建、打开、读写、关闭以及删除文件的服务
