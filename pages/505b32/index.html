07.设备管理

# 设备分类

  • 设备使用特性：

    存储设备、I/O设备、终端设备、脱机设备；

  • 从属关系：

    系统设备、用户设备

  • 资源管理：

    独占设备、共享设备、虚拟设备；

  • 信息的交换单位：

    块设备、字符设备

# 设备管理功能

  1. 设备分配

    按照设备类型和相应的控制算法，将设备分配给申请的进程，将未分配到设备的进程放入到等待队列；

  2. 设备处理

    提供进程管理系统的接口，当进程要求设备资源时，建立进程通信接口。

  3. 设备的独立性以及并行操作性

    为用户提供友好的透明借口，屏蔽了具体硬件设备的细节；

    具有极大的灵活性，易于实现I/O重定向；

  4. 缓冲管理

    CPU的执行速度和内存的访问速度较高，而设备的数据存取速度较低。系统通过建立缓冲区来实现数据存取的平衡，从而提高CPU的利用率；

# 1️⃣ 数据传送控制方式

# 程序直接控制方式

  • 由用户进程直接控制CPU和设备之间有相应的中断请求线内存或CPU和外围设备之间的信息传送；
  • 执行一个循环程序，反复查询外设状态，如果外设“忙碌”，则循环查询，知道为“闲置”时止；
  • 适用于没有中断机构的场合；
  • 优点：控制简单，不需要较多硬件支持；
  • 缺点：CPU和外围设备只能串行工作；CPU在一段时间内只能和一台外围设备进行数据交换，执行效率低；

# 中断方式

  • CPU和设备之间有相应的中断请求线；
  • 设备控制器的控制状态寄存器设置相应的中断允许位；
  • CPU与外设并行工作；
  • 适于低速外设I/O，并可配合DMA和通道方式实现I/O；
  • 优点：与程序直接控制方式相比，使CPU的利用率大大提高；支持多道程序和设备的并行操作；
  • 缺点：由于中断次数多，处理终端消耗大量CPU处理时间，系统开销大；并行的操作下，中断次数剧增可能造成CPU无法响应终端导致数据丢失；外围设备速度高，CPU来不及取走数据而丢失；

# DMA方式

  • 在外设和内存之间开辟直接数据交换通道；
  • 挪用CPU的一个工作周期，把数据缓冲寄存器中的数据直接送到地址寄存器所指向的内存区域中；
  • 适用于高速外设I/O，一次可传送一个或多个数据块（外设与内存之间），传输完毕才需要CPU干预；
  • 特点：与中断方式的主要区别：
    1. 中断方式是在数据缓冲寄存器满之后发中断要求CPU进行中断处理；
    2. DMA方式则是在所要求转送的数据块全部传送结束时要求CPU进行中断处理；
  • 缺点：当外围设备管理的控制越来越复杂，多个DMA控制器同时使用，会引起内存地址的冲突并使控制复杂化；

# 通道控制方式

通道：一个独立于CPU的专管输入输出控制的处理机，它控制设备与内存直接进行数据交换；

自己的通道指令，这些通道指令受CPU启动，并在操作结束时向CPU发中断信号；

通道指令：交换数据在内存中的位置、传送方向、数据块长度以及被控制的I/O设备的地址信息等；

通道方式与DMA方式的区别：

DMA 通道
数据控制 CPU 通道
设备对应关系 每台设备至少一个DMA控制器 一个通道控制多台设备
CPU负担 较大 较小

通道与一般的处理机的区别：

  1. 指令的类型单一；由于通道硬件比较简单，其所能执行的命令，主要局限于与I/O操作有关的指令；
  2. 通道没有自己的内存；通道所执行的通道程序是放在主机的内存中的，即通道与CPU共享内存；

四种I/O方式总结：

（1）程序I/O方式。在无中断的系统中使用，是忙－等方式； （2）中断驱动I/O控制方式。常用于字符设备； （3）直接存储器访问DMA控制方式。常用于块设备； （4）I/O通道控制方式。应用在高档小型、微型机中；
